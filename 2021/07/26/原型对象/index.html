JS原型对象

  • 每一个函数都有prototype属性，prototype就是函数的原型对象，称之为显式原型

  • 普通函数的prototype没有意义，只有在构造函数中才有意义

  • 原型对象默认是类名为Object的对象（Object的原型对象除外）

  • 原型对象中有一个constructor指向函数本身，而函数的prototype属性指向原型对象，形成循环指向

  • 以构造函数创建的每一个对象，都有一个隐含的属性__proto__指向构造函数的原型对象，称之为隐式原型，原型对象也有__proto__属性

  • 由同一个构造函数创建的对象，它们的__proto__属性都相同，即都指向构造函数的原型对象prototype

  • 类名为Object的对象，其原型对象的__proto__属性为null（原型链终点）

  • 类名不为Object的非函数对象，则对象.__proto__.__proto__.__proto__ == null

  • 原型链：本质是隐式原型链，当访问对象的属性和方法时

    • 先在对象自身中进行访问

    • 自身没有时则在原型对象中进行访问

    • 如果原型对象也有其原型对象，则接着进行访问

    • 如果最终没有访问到，则返回undefined

  • 如果只想访问对象自身的属性，可用hasOwnProperty(属性名)方法先判断对象自身是否具有该属性
  • hasOwnProperty(属性名)方法位于类名为Object的对象的原型对象中，即最后一层原型对象中
  • 任何函数(包括构造函数、Function)也属于Function实例对象，其也有隐式原型__proto__指向Function的显式原型，其通过构造函数Function创建，即new Function
  • Function本身也是Function实例对象，即Function = new Funtion()，Function的隐式原型和显示原型指向同一个原型对象
  • instanceof的语法A instanceof B，其中A是被检测对象，B是目标类(构造函数)，原理是B的显式原型是否在A的原型链上（Object的原型对象的隐式原型为null，故Object的原型对象不是Object对象）
  • typeof Function.prototype的结果是function，即Function的原型是一个函数对象
