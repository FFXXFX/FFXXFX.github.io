Promise

理解

  • Promise是es6中对于异步编程的解决方案
  • 语法上Promise是一个构造函数
  • 功能上Promise用来封装异步操作，并可以获得其成功/失败的结果值

作用

  • 由于Promise对象执行then时的返回值仍然是Promise对象，所以支持链式编程，可用来解决回调地狱问题
  • 指定回调函数的方式更加灵活

编码

  • 不使用Promise时
  • 多个异步操作的回调函数嵌套，便形成了回调地狱
  • 使用Promise来解决回调地狱，支持链式编程
  • 对读取文件操作进行封装成mineReadFile
  • 参数为文件路径path
  • 返回值为Promise对象
  • util.promisify(function)
    • 传入的是一个函数
    • 传入的函数的最后一个参数为错误优先的回调函数(err, value) => { 函数体 }
    • 返回值为一个新的函数
    • 返回函数的参数除没有回调函数外，与传入的函数保持一致
    • 返回的函数执行时的返回值是原先异步操作对应的Promise对象(形如上文封装的mineReadFile)

Promise的状态

  • Promise实例对象的[PromiseState]，初始值为pending

    • pending，未决定
    • resolved / fullfilled，成功
    • rejected，失败

  • Promise状态仅可改变一次

    • pending -> resolved / fullfilled
    • pending -> rejected

Promise的值

  • Promise实例对象的[PromiseResult]，保存着是对象状态是成功 / 失败的结果，即由resolve() / reject()传过来的结果

工作流程

API

构造函数Promise(executor)

  • executor函数称为执行器，形如(resolve, reject) => { 函数体 }
  • resolve函数为成功函数，内部定义成功时(即执行器中执行了resolve(value)时)，自动的异步调用onResolved函数
  • reject函数为失败函数，内部定义失败(即执行器中执行了reject(reason)）或者抛出异常时，自动的异步调用onRejected函数
  • executor函数会在Promise内部同步执行

Promise.prototype.then(onResolved, onRejected)

  • onResolved函数为成功时的回调函数，形如value => { 函数体 }
  • onRejected函数为失败时的回调函数，形如reason => { 函数体 }

Promise.prototype.catch(onRejected)

  • onRejected函数为失败时的回调函数，形如reason => { 函数体 }
  • 只接收失败状态的回调
